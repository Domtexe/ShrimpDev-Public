## Quelle der Wahrheit
- `modules/logic_actions.py` ist die **alleinige Quelle der Wahrheit** für:
  - Button → Action
  - Action → Runner
- UI-Komponenten rufen **Actions**, niemals direkt Runner.

## Gating-Prinzip
- Enable/Disable-Logik existiert **genau einmal pro Feature**.
- Beispiel:
  - Purge-Gating → `_update_purge_states()`
  - Keine doppelte Logik in Push-/anderen Tickers.
