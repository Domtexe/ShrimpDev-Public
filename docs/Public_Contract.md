# Public Contract

## Purpose

This public repository is a curated export of ShrimpDev project information.

It provides:
- a compact public overview
- a stable external-facing documentation surface
- public architecture and roadmap context

## Not Public

The public export intentionally does not include:
- internal pipeline planning
- internal governance and master rules
- reports, debug traces, audits, and aftercare history
- internal runtime storage, local state, journals, or machine-specific artifacts
- local machine paths
- internal working routines and operational process details

## Documentation Boundaries

Public docs should stay:
- concise
- technically useful
- understandable without internal workflow context

Public docs should avoid:
- internal runner IDs as work logs
- private repo structure details that are only operationally useful
- local workstation references
- internal decision noise and maintenance history

## External Reader Expectation

If a document is mainly useful for internal execution, governance, or maintenance, it should not be part of the public export.
