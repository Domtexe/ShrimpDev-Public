# Public Roadmap

## Current Focus

- keep ShrimpDev understandable for external readers and contributors
- maintain a clean public documentation surface
- document the product and architecture without exposing internal operational detail

## Near-Term Improvements

- clarify public-facing installation and usage documentation
- improve public architecture coverage for core subsystems
- keep the public export lean and easy to navigate

## Longer-Term Direction

- expand contributor-facing public guidance where it adds clear value
- refine public technical documentation as the product surface stabilizes
- keep public docs focused on product understanding rather than internal process

## Out of Scope for Public Roadmap

- internal runner planning
- internal governance and enforcement details
- private maintenance and incident workflows
