=======================================================

1. Bereich
   - Modul: modules.logic_actions
   - Funktion: action_tree_delete(app, *args, **kwargs)

2. Alt-Verhalten
   - Loeschaktion im Tree basierte auf ui_project_tree.get_selected_path(app).
   - Nur der erste selektierte Eintrag wurde geloescht.

3. Neues Verhalten (R2074)
   - Zuerst werden alle Pfade aus ui_project_tree.get_selected_paths(app)
     geholt. Falls leer, Fallback auf get_selected_path(app).
   - Jede selektierte Datei wird in den zentralen _Trash-Ordner im Projekt-Root
     verschoben (Soft Delete).
   - Fuer jede Datei wird ein Undo-Eintrag in app._tree_undo_stack erzeugt,
     mit action='delete', original=<Pfad>, trash=<Trash-Pfad>, time=<Timestamp>.
   - Am Ende wird, falls vorhanden, right_list.refresh() aufgerufen.

4. Konsequenz
   - Ein Delete-Klick im Tree wirkt nun auf alle markierten Dateien, nicht nur
     auf die erste.
   - Der Undo-Mechanismus kann wie bisher einzelne Loeschvorgaenge rueckgaengig
     machen, da jede Datei einen separaten Stack-Eintrag erhaelt.
