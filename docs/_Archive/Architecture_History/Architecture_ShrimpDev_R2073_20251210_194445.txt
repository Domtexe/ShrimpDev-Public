ShrimpDev Architektur-Notiz – R2073 (Tree-Button-Wiring)
========================================================

1. Bereich
   - Modul: modules.ui_project_tree
   - Funktion: wire_tree_buttons(app, wrap)

2. Problem vorher
   - Buttons für Delete/Rename/Undo wurden nur dann gefunden,
     wenn sie direkt in einem bestimmten Frame lagen.
   - Bei tieferen Layout-Verschachtelungen blieben die Buttons
     unverdrahtet, Aktionen hatten keine Wirkung.

3. R2073-Lösung
   - wire_tree_buttons sammelt nun rekursiv alle tk.Button-Widgets
     unterhalb von wrap.
   - Zuordnung primär über Button-Text (Löschen/Delete, Rename/Umbenennen,
     Undo/Rückgängig).
   - Fallback über Reihenfolge der Buttons, wobei Run/Start/Ausführen
     explizit ignoriert werden.
   - Commands werden auf logic_actions.action_tree_delete/rename/undo
     gesetzt.

4. Ergebnis
   - Delete/Rename/Undo funktionieren unabhängig vom genauen Layout
     des rechten Panels, solange die Buttons unterhalb von wrap liegen.
